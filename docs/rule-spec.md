# ルール仕様

**Microsoft Japanese Localization Style Guide** を参考に、外来語カタカナ語の末尾長音符（`ー`）の有無を統一する。

参考: [Microsoft - Localization Style Guides](https://learn.microsoft.com/en-us/globalization/reference/microsoft-style-guides)

## 検知方式

辞書方式（誤表記 → 正表記）による完全一致検知。辞書に収録されていない語は対象外。

## 基本規則

### 末尾の長音符を付ける: 語尾 -er/-or/-ar

英語原語の末尾が -er/-or/-ar で終わる単語は、カタカナ末尾の長音符を省略しない。

- 例: `ユーザー`（user）、`サーバー`（server）、`モニター`（monitor）

### 末尾の長音符の付け外し: 文字数による判定

-er/-or/-ar 以外の場合、末尾長音符を付けた形での文字数が3以下なら末尾長音符を保持し、4以上なら省略する。

文字数の数え方:
- 促音（ッ）、拗音（ャ・ュ・ョ）、小書き（ァ・ィ・ゥ）は文字数に数えない
- 長音符（ー）は1文字として数える

例: `ギルティー` -> ギ (1) + ル (1) + テ (1) + ィ (0) + ー (1) = **4** -> 4以上 → 末尾長音符なしが正 -> `ギルティ`

### 複合語（接頭語 + 語幹）の扱い

英語として接頭語 + 語幹の構造を持つ場合は、語幹部分を単独の語として上記原則を適用する。

- 例: `プレビュー`（preview = pre + **view**、語幹 `ビュー` は ビ (1) + ュ (0) + ー (1) = 2文字 → 3以下 → 末尾長音符ありが正）
  - 語全体で数えると4文字になるが、語幹単独では2文字のため末尾長音符を省略しない

### 慣用表記の考慮

上記原則に反する表記が慣用的に定着している場合は、慣用表記のほうを正表記として扱う。

- 例: `クリア`（clear）

## 適用範囲の制約

### 複合カタカナ語

本ルールはテキスト中のカタカナ連続列を1トークンとして辞書と完全一致照合する。
そのため、複合語全体を辞書登録しない限り、複合語の先頭または途中に誤りがある場合は検知できない。

例: `サーバリスト`（`サーバーリスト` が正しいが、`サーバ` と `リスト` の境界をテキストから特定できないため未対応）

これは仕様上の制約として許容する。ただし、偽陰性の発生頻度が高いと思われるケースについては、複合語の辞書登録も検討する。
