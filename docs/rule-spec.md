# ルール仕様

**Microsoft Japanese Localization Style Guide** を基本原則としつつ、より自然な日本語を目指した独自拡張ルールにより、外来語カタカナ語の末尾長音符（`ー`）の有無を統一する。

参考: [Microsoft - Localization Style Guides](https://learn.microsoft.com/en-us/globalization/reference/microsoft-style-guides)

## 検知方式

辞書方式（誤表記 → 正表記）による完全一致検知。辞書に収録されていない語は対象外。

## 基本規則

以下の規則を英語原語の語尾に基づいて順に適用する。

### 末尾の長音符を付ける: 子音 + -er/-or/-ar

英語原語の末尾が子音 + -er/-or/-ar で終わる単語は、カタカナ末尾の長音符を省略しない。

- 例: `ユーザー`（user）、`サーバー`（server）、`モニター`（monitor）

母音 + -er/-or/-ar（-ear, -ier, -oor 等）はこの規則の対象外であり、末尾の長音符を付けない。

- 例: `ビヘイビア`（behavior）、`アンプリファイア`（amplifier）、`リニア`（linear）

### 末尾の長音符を付ける: -y（-ry, -ty, -phy を除く）

英語原語の末尾が -y で終わり、かつ -ry, -ty, -phy のいずれでもない単語は、カタカナ末尾の長音符を省略しない。

- 例: `ポリシー`（policy）、`テクノロジー`（technology）、`タクソノミー`（taxonomy）

### 末尾の長音符を付けない: -ry

英語原語の末尾が -ry（-ory, -ary を含む）で終わる単語は、カタカナ末尾の長音符を省略する。

- 例: `メモリ`（memory）、`ディレクトリ`（directory）、`ライブラリ`（library）

### 末尾の長音符を付けない: -ty / -phy

英語原語の末尾が -ty（-bility, -ity を含む）または -phy で終わる単語は、カタカナ末尾の長音符を省略する。

- 例: `プロパティ`（property）、`セキュリティ`（security）、`タイポグラフィ`（typography）

### 末尾の長音符を付けない: -ure

英語原語の末尾が -ure で終わる単語は、カタカナ末尾の長音符を省略する。

- 例: `アーキテクチャ`（architecture）、`クロージャ`（closure）

### 複合語（接頭語 + 語幹）の扱い

英語として接頭語 + 語幹の構造を持つ場合は、語幹部分を単独の語として上記規則を適用する。

- 例: `プレビュー`（preview = pre + **view**）— 語幹 view は子音 + -er/-or/-ar でも -y でもないが、慣用的に長音符ありが定着

### 慣用表記の考慮

上記規則に反する表記が慣用的に定着している場合は、慣用表記のほうを正表記として扱う。

- 例: `コンパイラ`（compiler — 子音 + -er で規則上は長音符ありだが、慣用的に長音符なし）

## 適用範囲の制約

### 複合カタカナ語

本ルールはテキスト中のカタカナ連続列を1トークンとして辞書と完全一致照合する。
そのため、複合語全体を辞書登録しない限り、複合語の先頭または途中に誤りがある場合は検知できない。

例: `サーバリスト`（`サーバーリスト` が正しいが、`サーバ` と `リスト` の境界をテキストから特定できないため未対応）

これは仕様上の制約として許容する。ただし、偽陰性の発生頻度が高いと思われるケースについては、複合語の辞書登録も検討する。
