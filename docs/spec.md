# ルール仕様

外来語カタカナ語の末尾長音符（ー）の表記を **Microsoft Japanese Localization Style Guide** に準拠した形に統一する。

参考: [Microsoft - Localization Style Guides](https://learn.microsoft.com/en-us/globalization/reference/microsoft-style-guides)

## 検知方式

辞書方式（誤表記 → 正表記）による完全一致検知。辞書に収録されていない語は対象外。

## 基本規則

### 末尾の長音符を付ける: -er/-or/-ar 語尾

英語原語の末尾が **-er / -or / -ar** で終わる単語は、カタカナ末尾の長音符を省略しない。

- 例: `ユーザー`（user）、`サーバー`（server）、`モニター`（monitor）

### 末尾の長音符の付け外し: 文字数による判定

-er/-or/-ar 以外の場合、**末尾に長音符を付けた形での文字数が3以下** なら付け、**4以上** なら省略する。

文字数の数え方:

| 種別 | 例 | 文字数への算入 |
|------|----|:---:|
| 通常のカタカナ | ア、コ、ン … | ✓ |
| 長音符 | ー | ✓ |
| 促音 | ッ | × |
| 拗音 | ャ・ュ・ョ | × |
| 小書きアイウ | ァ・ィ・ゥ | × |

例:

| 語 | 内訳（末尾長音符付き形） | 判定 |
|----|---------------------|------|
| `キー`（key） | キ (1) + ー (1) = **2** | 3以下 → 末尾長音符付き `キー` |
| `メニュー`（menu） | メ (1) + ニ (1) + ュ (0) + ー (1) = **3** | 3以下 → 末尾長音符付き `メニュー` |
| `メモリー`（memory） | メ (1) + モ (1) + リ (1) + ー (1) = **4** | 4以上 → 末尾長音符省略 `メモリ` |
| `ディレクトリー`（directory） | デ (1) + ィ (0) + レ (1) + ク (1) + ト (1) + リ (1) + ー (1) = **6** | 4以上 → 末尾長音符省略 `ディレクトリ` |

### 複合語（接頭語 + 語幹）の扱い

英語として接頭語 + 語幹の構造を持つ場合は、語幹部分を単独の語として上記規則を適用する。

- 例: `プレビュー`（preview = pre + **view**、語幹 `ビュー` は ビ (1) + ュ (0) + ー (1) = 2文字 → 3以下 → 末尾長音符付き）
  - 語全体で数えると4文字になるが、語幹単独では2文字のため末尾長音符を省略しない

### 慣用表記の考慮

上記規則に反する表記が慣用的に定着している場合は、慣用表記のほうを正表記として収録するか、または両方許容するべく収録対象外とする。

- 例: `クリア`（clear、-ear 語尾）

## 適用範囲

### 複合カタカナ語

本ルールはテキスト中のカタカナ連続列を1トークンとして辞書と完全一致照合する。
そのため、複合語全体を辞書登録しない限り、複合語の先頭または途中に誤りがある場合は検知できない。

例: `サーバリスト`（`サーバーリスト` が正しいが、`サーバ` と `リスト` の境界をテキストから特定できないため未対応）

これは仕様上の制約として許容する。ただし、偽陰性の発生頻度が高いと思われるケースについては、複合語全体の辞書登録も検討する。
